Introduction

Nos API de prédiction vous permettent d'accéder aux données en temps réel des jeux pour intégrer des fonctionnalités de prédiction dans vos applications.

Toutes nos API utilisent le même endpoint avec des headers spécifiques pour chaque jeu.

Configuration de base

Pour utiliser nos API, vous devez inclure les headers suivants dans vos requêtes :

const apiURL = "https://crash-gateway-grm-cr.100hp.app/state";
const headers = {
  'customer-id': '077dee8d-c923-4c02-9bee-757573662e69',
  'session-id': 'SESSION_ID_DU_JEU',
  'accept': 'application/json',
};

Le customer-id est le même pour tous les jeux, tandis que le session-id varie selon le jeu.

Exemple d'intégration

Voici un exemple complet d'intégration de l'API pour récupérer les coefficients d'arrêt :

// Configuration de l'API
const apiURL = "https://crash-gateway-grm-cr.100hp.app/state";
const headers = {
  'customer-id': '077dee8d-c923-4c02-9bee-757573662e69',
  'session-id': '95a987ef-72e1-4eaa-a598-64a157a44e75', // Exemple pour Crash
  'accept': 'application/json',
};

let coefficientsHistory = [];

// Fonction pour récupérer le coefficient d'arrêt
async function fetchStopCoefficient() {
  try {
    const response = await fetch(apiURL, { headers });
    if (!response.ok) throw new Error("Erreur API");

    const data = await response.json();
    return data.stopCoefficients?.[0] ?? null;
  } catch (error) {
    console.error("Erreur :", error);
    return null;
  }
}

Bonnes pratiques

  • Implémentez une gestion d'erreurs robuste pour gérer les interruptions de service
  • Utilisez des requêtes asynchrones pour ne pas bloquer l'interface utilisateur
  • Stockez les données historiques pour analyser les tendances
  • Limitez la fréquence des requêtes pour éviter de surcharger le serveur
  • Testez votre intégration avec différents jeux pour valider le comportement
